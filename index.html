广告

游戏介绍

在由方块构成的无限世界中，创造属于你的故事

🌍

开放世界

每次生成的世界都是独一无二的。平原、沙漠、雪原、丛林、深暗之域……生物群系丰富多样，等待你探索。

⛏️

采集与合成

用手破坏方块获取资源，在工作台中合成工具、武器与建筑材料。从木镐到下界合金装备，逐步升级你的能力。

🏠

自由创造

建造从小屋到城堡、从自动农场到红石计算机，几乎没有限制。单人沉浸或多人联机，玩法由你定义。

游戏模式

四种模式，满足不同玩家的需求

生存模式 Survival

最核心的玩法。你需要管理生命值与饥饿值，采集资源、合成物品、躲避或战斗怪物。死亡会掉落物品（除非设置了 keepInventory）。

  • 有限资源，需要规划与探索
  • 昼夜交替，夜晚怪物出没
  • 可挑战下界与末影龙
  • 支持成就/进度系统

创造模式 Creative

拥有无限资源，可飞行，不会受到伤害。适合建筑创作、红石实验和地图设计，无需担心生存压力。

  • 物品栏中所有方块与物品无限
  • 双击空格键飞行
  • 瞬间破坏方块
  • 无法获得成就

冒险模式 Adventure

专为自定义地图设计。玩家不能随意破坏方块，只能使用特定工具破坏特定方块，适合剧情类地图与解谜地图。

  • 方块破坏受 NBT 标签限制
  • 可与命令方块配合使用
  • 常用于服务器小游戏

旁观模式 Spectator

以幽灵形态观察世界，可穿墙飞行，无法与方块或实体交互。常用于观战或地图调试。

  • 可穿墙自由移动
  • 点击实体可附身观察
  • 无法影响游戏世界

新手入门

第一天的生存步骤，按顺序完成即可安全过夜

1

收集木头

找到树木，用手或快速破坏树叶与原木。至少收集 6~10 个原木，这是所有起步工具的基础。

2

制作工作台与木镐

将原木放入背包合成面板变为木板，4 木板做工作台。制作木镐后，挖 3 圆石 升级石镐。

3

搭建临时庇护所

在日落前用泥土或圆石围一个 3×3 空间，封顶并留门。放置 火把（煤炭+木棍）照明，防止怪物生成。

4

解决食物问题

击杀牛、猪、羊获得生肉；或寻找村庄拿面包。优先制作 熔炉 烤肉，熟肉回复更多饥饿值。

5

第二天：下矿找铁

携带火把、石镐、食物，在 Y=-16 到 Y=32 层寻找铁矿石。挖到 16+ 铁锭 后制作铁镐、铁剑、盾牌。

战斗

常见怪物应对

  • 僵尸：普通近战，注意小僵尸速度快。用盾牌格挡后反击。
  • 骷髅：远程攻击，之字形靠近或使用盾牌。白天可在阴影下攻击。
  • 苦力怕：靠近会嘶嘶作响并爆炸。听到声音后立即后退 4 格以上，或用水流阻挡。
  • 末影人：不要直视眼睛。戴南瓜头可安全观察，或站在 2 格高空间下单挑。
  • 凋灵：后期 Boss，需准备下界之星、药水与大量金苹果。
附魔

附魔系统入门

使用经验等级在附魔台为装备附加特殊能力。附魔台需要周围放置书架（最多 15 个，间隔 1 格）来提升等级上限。

  • 保护 IV：通用防御附魔，优先给胸甲
  • 效率 V：加速挖掘，给镐子
  • 锋利 V：增加近战伤害
  • 时运 III：增加矿物掉落（钻石矿必附）
  • 经验修补：消耗经验修复耐久，后期神器

💡 村民交易是获取特定附魔书的捷径，搭建村民繁殖与交易大厅可大幅节省时间。

农业

食物与农场

用锄头耕地，附近要有水源，种植小麦、胡萝卜、马铃薯。骨粉可加速生长。

  • 动物繁殖：用对应食物（小麦/胡萝卜/种子）让动物进入爱心状态
  • 自动农场：利用活塞、观察者、漏斗实现自动收割
  • 村民农场：农民村民可自动种植与收获作物
  • 下界疣：在下界用灵魂沙种植，用于酿造药水

建造技巧

让建筑更美观、更实用的设计原则

🎨 材质搭配

避免单一材质。组合使用不同木材、石头、陶瓦与混凝土，增加层次感。参考真实建筑的比例，窗户不要过大。

💡 照明与细节

使用灯笼、火把、海晶灯、发光浆果等不同光源。添加楼梯、台阶、活板门、花盆等装饰方块打破大平面的单调感。

🏔️ 地形融合

建筑应顺应地形而非强行平整。在 hillside 采用分层结构，在水边加入码头与栈桥，让作品"长"在环境里。

📐 功能分区

生存基地建议分区：存储区（大量箱子+分类）、农牧区、挖矿入口、附魔室、下界传送门。用铁路或冰道连接各区域。

进阶内容

下界、末地、红石——挑战更高难度的游戏内容

🔥

下界攻略

  1. 用 10+ 黑曜石搭建传送门（4×5 框架），打火石激活
  2. 穿戴金盔甲防止猪灵攻击（不要挖他们的金块）
  3. 寻找下界要塞获取烈焰棒，用于酿造与末地之眼
  4. 在 Y=8~22 用 TNT 或床爆破法寻找远古残骸
  5. 4 金锭 + 1 远古残骸 → 下界合金锭，升级钻石装备
🐉

末地 · 击败末影龙

  1. 收集 12 个末影之眼，找到要塞并激活传送门
  2. 准备：钻石/下界合金装备、弓箭、大量箭、金苹果、缓降药水
  3. 先摧毁所有末地水晶（塔顶），防止龙回血
  4. 用弓箭射击或等待龙俯冲时用剑攻击
  5. 击败后探索末地城，获取鞘翅与潜影盒
⚡

红石基础

  • 红石粉：传递 15 格信号，需中继器延长
  • 比较器/中继器：延迟、比较、锁存信号
  • 活塞/观察者：自动农场与隐藏门核心
  • 漏斗：物品传输与自动分类系统
  • T _flip_flop：用按钮实现开关切换，入门必学

开源项目

Steve — 自主 AI 代理模组

Steve 由 Yuvraj Dwivedi 与 Pravin Lohani 开发，将大语言模型（LLM）接入 Minecraft， 让 AI 代理像真实玩家一样在世界中执行指令。你描述需求，Steve 理解上下文、制定计划并逐 tick 执行—— 不是写代码的 AI，而是在游戏里帮你干活的 AI。

核心能力

⛏️

智能采集

说「挖一些铁」，代理会推理铁矿分布层级、寻路到合适深度、定位矿脉并开采，无需手写脚本。

🏗️

自主建造

根据可用材料生成结构方案，逐方块搭建房屋、平台、城堡等，支持程序化生成与 NBT 模板加载。

🤝

多代理协作

多个 Steve 可并行施工：自动划分区域、分配任务、避免冲突，例如三人同时建一座城堡。

安装与配置

环境要求

  • Steve 1.4.1（推荐）+ Minecraft 1.20.1 + Forge 47.2.0
  • Java 17（1.20.1–1.20.4）；1.20.5+ 需 Java 21
  • LLM API Key（OpenAI / Groq / Gemini 任选）

安装步骤

  1. 从 Modrinth 下载 1.4.1 或 GitHub Releases 获取 JAR，放入 mods 文件夹
  2. 启动游戏一次，生成配置目录
  3. 复制 config/steve-common.toml.example 为 config/steve-common.toml
  4. 填入 API Key 与模型配置
  5. 进游戏执行 /steve spawn Bob 召唤代理
  6. 按 K 打开指令面板，输入自然语言命令

config/steve-common.toml 示例

[llm]
provider = "groq"  # openai | groq | gemini

[openai]
apiKey = "sk-..."
model = "gpt-3.5-turbo"
maxTokens = 1000
temperature = 0.7

[groq]
apiKey = "gsk_..."
model = "llama3-70b-8192"
maxTokens = 1000

[gemini]
apiKey = "AI..."
model = "gemini-1.5-flash"
maxTokens = 1000

💡 游戏内推荐用 Groq（响应快）；复杂规划可试 GPT-4。temperature 建议 0.5~0.7，行为更稳定。

无需精确语法，代理会自行理解意图并拆解为移动、挖矿、放置、战斗等动作序列。

游戏命令

常用 /steve 命令

命令说明
/steve spawn <名字>召唤一名 Steve 代理
K打开指令面板，输入任务

每个 Steve 拥有独立记忆与任务上下文，可给不同代理分配不同工作。

结合本攻略的实战场景

🌱 生存初期

  • "gather 20 oak logs" — 替代手动砍树，快速完成第一天物资准备
  • "build a small shelter near me with a door" — 日落前自动搭临时庇护所
  • "place torches around this area" — 批量照明，防止怪物生成

⛏️ 资源采集

  • "mine iron ore until you have 32" — 配合上文「分支挖矿法」，专注探索与基地规划
  • "dig down to Y -54 and mine diamonds" — 代理按层级寻找钻石（需铁镐以上工具，见合成指南）
  • "collect cobblestone and fill this chest" — 自动化收集建材

🏠 基地建造

  • "build a storage room with 10 chests" — 快速搭建存储区
  • "build a 9x9 wheat farm" — 配合农牧攻略，解放重复劳动
  • 召唤 2~3 个 Steve："build a castle together" — 体验多代理并行施工

⚔️ 战斗与探索

  • "defend me from zombies tonight" — 夜间守家，你专注合成与附魔
  • "follow me and attack hostile mobs" — 下界/洞穴探索时的护卫
  • "light up this cave" — 探索前让代理先行铺设火把

2

TaskPlanner 调用 LLM 生成动作计划

→
3

ActionExecutor 逐 tick 执行（挖矿/建造/移动/战斗）

→
4

采用「直接动作执行」而非传统 ReAct 循环，减少 API 调用与延迟，保证游戏不卡顿。多代理协作由服务端 CollaborativeBuildManager 做空间分区与冲突避免。

注意事项

当前有哪些局限？

1.4.0 之前版本无法合成工具；1.4.0+ 虽引入合成功能（CraftItemAction），但仍处于早期阶段，成功率取决于 LLM。单任务同步执行，挖矿时无法并行其他操作；重启后记忆会重置。智能程度取决于所选 LLM，GPT-3.5 偶有不合理决策，GPT-4 规划更好但更慢。

如何从源码构建？

克隆仓库后执行 ./gradlew build，产物在 build/libs/。开发调试可用 ./gradlew runClient。项目结构包含 llm/、action/、structure/、memory/ 等模块，MIT 开源，欢迎提交 Issue 与 PR。

API 费用如何控制？

每次指令会调用 LLM，频繁下达复杂任务会产生 API 费用。建议先用 Groq 免费额度测试；简单任务用短句；多代理协作时注意同时发起的请求数量。勿将含 API Key 的配置文件上传到公开仓库。

常见问题

新手应该选什么难度？

第一次玩建议选择「普通」难度。若觉得战斗压力大，可暂时设为「和平」（无怪物），专注熟悉采集与建造，之后再调回普通。

如何备份存档？

单人存档位于 .minecraft/saves/ 文件夹，复制整个世界文件夹即可备份。多人服务器需备份 world 目录及 server.properties。

原版和模组有什么区别？

原版是 Mojang 官方内容。模组（Mod）由社区开发，可添加新生物、维度、科技系统等。整合包是预装多个 Mod 的模组组合，玩法变化更大。

怎么和朋友一起玩？

Java 版可通过 LAN 开放、Realms 订阅或自建/租用服务器联机。基岩版支持跨平台，通过好友邀请或 Realms 即可。
